# 课程大纲优化说明

## 📋 优化概述

根据 `docs/enterprise/` 目录下的系统化文档结构，对原课程大纲进行了重大优化，使其更符合企业级项目开发流程。

---

## 🔄 主要变化

### 1. 课程结构重组

**原结构**：
```
- 任务 12: 架构设计 (40分钟)
- 任务 13: 全栈开发 (80分钟)
- 任务 14: Docker 容器化 (40分钟)
```

**新结构**：
```
- 任务 12: 需求与设计 (60分钟)
  ├─ 需求调研与分析 (15分钟)
  ├─ 系统架构设计 (20分钟)
  ├─ 数据模型设计 (15分钟)
  └─ 业务流程设计 (10分钟)
  
- 任务 13: 全栈开发 (80分钟)
  ├─ Part 1: 后端开发 (40分钟)
  └─ Part 2: 前端开发 (40分钟)
  
- 任务 14: Docker 容器化 (40分钟)
```

## 💡 使用建议

### 对于讲师
1. **授课前准备**：
   - 先阅读 `docs/enterprise/` 下的所有文档
   - 理解企业级开发的完整流程
   - 准备好架构图、流程图、ER 图

2. **授课顺序**：
   - 严格按照：需求 → 架构 → 数据 → 流程 → 开发 → 部署
   - 每个阶段都要强调输出物和检查点

3. **实战演示**：
   - 现场演示如何从需求调研开始，逐步推导出架构设计
   - 展示如何根据数据模型设计 API 接口
   - 演示容器化部署的完整流程

### 对于学员
1. **学前准备**：
   - 了解 FastAPI、Streamlit、LangGraph 基础
   - 熟悉 Docker 基本命令
   - 准备好 OpenAI API Key

2. **学习路径**：
   - 第一遍：跟随课程完成所有实战
   - 第二遍：深入阅读 `docs/enterprise/` 文档
   - 第三遍：尝试扩展功能（选做作业）

3. **检验标准**：
   - 能够独立完成需求调研和架构设计
   - 能够从零搭建一个企业级 AI 系统
   - 能够使用 Docker 一键部署到生产环境
