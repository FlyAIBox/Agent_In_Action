## 📖 使用指南

### 对于讲师

**准备工作**：
1. 阅读所有 `enterprise/` 下的参考文档
2. 理解企业级开发的完整流程
3. 准备架构图、流程图、ER 图

**授课顺序**：
1. 严格按照：需求 → 架构 → 数据 → 流程 → 开发 → 部署
2. 每个阶段强调输出物和检查点
3. 结合实际代码进行演示

**课程节奏**：
- 任务 12（需求与设计）：60分钟，重在设计思维培养
- 任务 13（全栈开发）：80分钟，重在代码实现能力
- 任务 14（容器化）：40分钟，重在部署运维能力

**学前准备**：
- [ ] 了解 FastAPI、Streamlit、LangGraph 基础
- [ ] 熟悉 Docker 基本命令
- [ ] 准备好 OpenAI API Key
- [ ] 克隆项目代码到本地

**学习路径**：

**第一遍**（跟随课程）：

3. 学习数据模型设计

1. 阅读 `enterprise/` 所有文档
2. 理解每个设计决策的原因
3. 分析代码结构和实现细节

**第三遍**（扩展实践）：

2. 尝试选做作业

4. 优化性能和安全性

**检验标准**：
- [ ] 能够独立完成需求调研

## 💡 常见问题

### Q1: 为什么要从需求调研开始？
**A**: 企业级项目开发必须从需求开始，才能确保：
- 开发方向正确，避免返工
- 架构设计有据可依
- 功能设计符合业务场景
- 验收标准明确

### Q2: 数据模型设计为什么重要？
**A**: 数据模型是系统的基础：
- 决定了 API 接口的设计
- 影响前后端的对接
- 关系到数据库的设计
- 影响系统的可扩展性

### Q3: 为什么要学习业务流程设计？
**A**: 业务流程设计帮助你：
- 理解系统的完整运行逻辑
- 识别潜在的异常场景
- 设计合理的错误处理
- 优化用户体验

### Q4: 如何快速上手？
**A**: 建议按以下步骤：
1. 先看 `课程大纲-优化版.md` 了解全貌
2. 跟随课程完成第一遍实战
3. 深入阅读 `enterprise/` 文档
4. 尝试修改和扩展功能

### Q5: 遇到问题如何排查？
**A**: 参考以下资源：
1. 查看 `课程大纲-优化版.md` 第五部分（故障排除）
2. 阅读 `enterprise/06_deployment_and_operations.md`
3. 检查日志文件 `logs/backend.log`
4. 使用健康检查接口 `/health`
