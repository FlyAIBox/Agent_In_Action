## Slide 5: OpenAI Operator：AI 开始"动手"了

**标题：** 从对话到操作，AI Agent 的能力边界在哪里？

**Operator 能做什么？**

1. **浏览器自动化**
   - 打开网页、填写表单、点击按钮
   - 预订餐厅、下单购物、查询信息
   - 多步骤任务自动完成

2. **视觉理解 + 操作执行**
   - 看懂页面布局（计算机视觉）
   - 理解任务意图（NLP）
   - 执行操作指令（RPA）

3. **人机协同**
   - 遇到复杂情况请求人类确认
   - 从人类反馈中学习优化

**技术原理拆解：**

```
用户指令："帮我预订明晚7点的餐厅"
    ↓
AI 任务规划：
  1. 打开订餐网站
  2. 搜索附近餐厅
  3. 筛选明晚7点有空位的
  4. 选择评分最高的
  5. 填写预订信息
  6. 确认订单
    ↓
工具调用层：
  - 浏览器控制工具（Selenium/Playwright）
  - 视觉识别工具（GPT-4V）
  - 决策引擎（LangGraph）
    ↓
执行 + 反馈循环
```

**对我们的启发：**
- **核心不是"模型多强"，而是"工具调用"**
- **第1周我们就会学习工具调用的完整机制**
- **第3周构建的旅行智能体就是 Operator 的垂直领域版**

**演讲要点：**
- 播放 Operator 演示视频（30秒）
- 强调：技术原理我们都会学到
- 这不是魔法，而是工程化的技术栈

## Slide 8: 智能体开发的"冰山模型"

**标题：** 水面以上 vs 水下隐藏的复杂度

**水面以上（看起来简单）：**
```python
# 10行代码实现智能助手？
from openai import OpenAI
client = OpenAI()
response = client.chat.completions.create(
    model="gpt-4",
    messages=[{"role": "user", "content": "你好"}]
)
print(response.choices[0].message.content)
```

**水下隐藏的复杂度：**

1. **工具集成层**
   - 如何标准化工具接入？（MCP 协议）
   - 如何处理工具调用失败？
   - 如何优化工具调用成本？

2. **多智能体协作层**
   - 如何设计角色分工？
   - 如何管理复杂状态？
   - 如何实现人机协同？

3. **工程化实践层**
   - 如何容器化部署？
   - 如何监控性能和质量？
   - 如何持续优化迭代？

4. **垂直领域定制层**
   - 如何微调领域模型？
   - 如何构建高质量数据集？
   - 如何评估微调效果？

**能力层级图：**

```
Level 5: 垂直领域架构师 🏆
         ↑ 模型微调、私有化部署
Level 4: 企业级工程师 💼
         ↑ 容器化部署、监控评估
Level 3: 多智能体开发者 🤖
         ↑ LangGraph、状态管理
Level 2: 工具集成工程师 🔧
         ↑ Function Call、MCP 协议
Level 1: API 调用者 👶
         ↑ 会调用 OpenAI API
```

**本课程的目标：**
- **入学水平：** Level 1-2（会 Python 和基本 API 调用）
